# Step 3: Productivity & Task Efficiency Definition

## MANDATORY EXECUTION RULES:
- 🛑 NEVER generate content without user input
- 💬 FOCUS on "Time to Task Completion" and "Information Density"
- 🎯 COLLABORATIVE discovery for internal power users

## PRODUCTIVITY DISCOVERY SEQUENCE:

### 1. Define Primary Task Loop
"For internal tools, the core loop is often a repetitive data task. 
- What is the high-frequency action for your primary internal role?
- How can we make this 'zero-friction'?"

### 2. Information Hierarchy
"Internal users often need more data than external users.
- Where do we need high-density tables?
- What data is 'at-a-glance' vs 'drill-down'?"

### 3. Error Prevention Patterns
"How does the UI prevent internal process failures?
- Inline validation?
- Confirmation modals for destructive actions?
- Audit trail visibility?"

### 4. Experience Principles (Internal)
"I'm hearing these productivity principles:
- **Principle 1:** Efficiency over Aesthetics (Speed first)
- **Principle 2:** Data Integrity (Prevent entry errors)
- **Principle 3:** Predictability (Standard internal patterns)"

### 5. Generate Productivity Content
Append to document:
```markdown
## Productivity & Task Efficiency

### Core Workflow Loops
[Definition of the primary actions for internal roles]

### Information Density Strategy
[Approach to tables, filters, and data-heavy views]

### UX Guiding Principles
[List of internal-focused principles]
```
